The toast was the visible half; the tool was really being re-set. Root cause is uitk state restore, not the slot: widgets persist to QSettings by default (`restore_state=True`), and restoring a value **fires the slot** — `StateManager.apply` *unblocks* signals per `block_signals_on_restore`, whose default `False` deliberately overrides `init_slot`'s outer `blockSignals(True)` — so a stored "Marquee" re-entered `chk005` on panel open and ran `set_viewport_tool` (toast included).
