# tentacle — Changelog

## 2026

- **2026-06-03 Feature/Bugfix** — The **Deformation panel** (`deformation.ui`) was a dead, empty stub: the switchboard resolves UI `deformation` → slot class `DeformationSlots` (via `get_slot_class_names`), but no such class existed (the only deform-related slot, `deform.py DeformSlots`, binds to Maya's native *Deform* menu — UI key `deform` — not the panel). Added `slots/maya/deformation.py` (`DeformationSlots`) so the panel finally binds slots, and wired its `tb001` button as a **Curtain Generator** launcher (`tb001_init` labels it; `tb001` → `marking_menu.show("curtain")`) opening the new mayatk Curtain tool — a procedural pleated, gravity-draped (catenary) curtain hung from a selected rail curve / edges / locators (or a generated arc).
- **2026-06-03 Feature** — Materials module gained two more external-tool launchers alongside the Metashape one: **RealityCapture Workflow** (`b024` → `external_app.launch("realityscan_workflow")`) and **Brush Splat Workflow** (`b025` → `external_app.launch("gaussian_splat_workflow")`), both added to the Materials submenu's **External** Tools list. Registered in `tcl_maya.py` with their explicit subpackage modules (`extapps.photogrammetry.realityscan_workflow:RealityScanWorkflowUI`, `…gaussian_splat_workflow:GaussianSplatWorkflowUI`), matching the new `extapps` entry points (pinned by `test_package.py::TestExtappsRegistration`). The panels themselves (same `uitk.bridge` style as Metashape, on a shared photogrammetry-panel base) live in `extapps`.
- **2026-06-02 Feature** — `materials.py cmb002` (current material combo) gained an option box (a dropdown separate from its right-click context menu) with a **Hide Default Materials** toggle, off by default. The combo now passes `exclude_defaults=<toggle>` to `MatUtils.get_scene_mats` and no longer hardcodes `exc="standardSurface"` — so by default all materials show (including Maya defaults like `lambert1`/`standardSurface1` and user-created `standardSurfaceN`), and enabling the toggle hides only the true built-in defaults while keeping user materials. Toggling re-populates the combo via `init_slot`. The **Cleanup** actions (`b014` Remove Duplicate Materials, `b015` Delete All Unused Materials) also moved out of the right-click context menu into this option box (as pushbuttons under a "Cleanup" separator); the context menu now carries only the Edit and View/Select sections. Verified under mayapy standalone (`get_scene_mats` shows/hides defaults as expected; user `standardSurfaceN` always present); existing `test_materials.py` unit tests still pass.
- **2026-06-02 Bugfix** — `tcl_maya.py` registered every extapps app with `module=f"extapps.{_name}"`, but `metashape_workflow` lives in a subpackage (`extapps.photogrammetry.metashape_workflow`). The derived path was wrong and **overwrote** the correct one auto-discovery had registered (the manual call wins, carrying `install_spec`), so launching the Metashape panel hit `_is_importable() == False` → surprise `pip install` → *"Install of 'extapps' completed but 'extapps.metashape_workflow' is still not importable"*. Fixed by spelling out each app's `module` explicitly in the registration table instead of deriving it from the name (the name→module mapping isn't 1:1 once an app moves into a subpackage). uitk's `ExternalAppHandler.launch()` self-heal masked this whenever the entry-point metadata was present and correct, but not on a machine where `extapps` wasn't yet installed — this fixes the root cause. Verified in a fresh Maya (`mayatk.MayaConnection`): the corrected registration is importable on the first check (no self-heal needed) and the panel constructs.
- **2026-05-19 Rename** — `ExternalToolHandler` → `ExternalAppHandler` in uitk. Tentacle adapted: handler key `external_tool` → `external_app` in `tcl_maya.py` (registration) and in all slot call sites (`materials.py` b008/b016/b022/b023, `scene.py` b013). Naming now matches the `extapps` package.
- **2026-05-19 Refactor** — Migrated `materials.py b008` (Map Packer), `materials.py b016` (Map Converter), and `scene.py b013` (Mesh Convert) off the direct `from extapps import ...` pattern and onto `external_tool.launch(name, show=False)`. The three slots previously crashed with `ImportError` when extapps was not installed, bypassing the install-on-demand contract that `b022`/`b023` already honored. All five extapps tools now route through `ExternalToolHandler` uniformly. Removed redundant per-slot styling — the launcher's `__new__` already applies it.
- **2026-05-11 Bugfix** — `edit.py tb001` (Delete History → Delete Unused Nodes) was deleting instanced objects. Empty-group sweep used `cmds.ls` without `allPaths=True`, so only one DAG path per instanced node was listed; sibling instance transforms then appeared in `groups` but not in `dag_parents` and were deleted. Added `dag=True, allPaths=True` to the three `cmds.ls` calls that build `all_dag`, `exact_transforms`, and `shape_parents`. Regression test in `test/test_edit.py` (shape instance, transform instance, truly-empty group).
- **2026-03-26 Bugfix** — `ExpandableList list000` (Create Primitive) double-execution. `eventFilter` wasn't consuming `MouseButtonRelease` after emitting `on_item_interacted`, allowing re-trigger. Added `return True` after emission in `uitk/widgets/expandableList.py`.
- **2026-03-03 Perf** — `edit.py tb001` (Delete History) optimized for heavy scenes. Added AST regression tests in `test_slot_integrity.py`.
